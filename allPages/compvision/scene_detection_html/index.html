Scene Recognition with Bag of Words

This project implements scene recognition, a part of image recognition. Different techniques were used to represent the images and then classify them. A collection of 1500 images were used as a pool of training data for 15 different scene categories ranging from kitchens to highways to forests. The accuracy of this model was then evaluated on test images with the more sophisticated techniques having an accuracy of 68.8% and the faster, less intricate methods having accuracy of only 21.7% (for comparison purposes, random guessing for 15 categories would yield a rate of 7%).

Step 1) Image Representation

For image representation, I use two different techniques: tiny image representation and bag of SIFT features. The tiny image representation involves resizing the images to a 16x16 image and then using those pixels as descriptors. I was able to improve accuracy by 3% by using the zero mean of this image and then normalizing the final descriptor. These descriptors are very easily calculated and work very fast, however their simplistic nature means they have a limited effect on increasing accuracy.

Getting a bag of sift features consisted of two parts: building a vocabulary of words by clustering the image features based on the number of words desired and then sorting input images based on these words. In order to buld my vocabulary, I extracted the SIFT Features from images using a large step size. I then used k-means clustering, which groups these features together into clusters of a prescribed size. I found that having a vocab size of 400 words was ideal. I tried increasing the vocab size and found that it would only marginally improve accuracy while greatly increasing runtime, so I decided that it was not worth the time cost. Once I had my vocabulary, I extracted SIFT features from input images using a small step size of 3. I then sorted these into a histogram based on their closest match in the training features. The histogram contains votes based on how many SIFT descriptors fell into the cluster for that word. The normalized histogram then became the image feature descriptor.The bag of words model downplays any spatial information because it disregards where in an image a feature occurs. To increase accuracy, spatial information could be taken into consideration.

Step 2) Classification techniques

In classification, I used both nearest neighbor classifier and SVM classify. For nearest neighbor classsification, I found the best match by voting based on the 11 nearest neighbors, which is found by comparing the distance of the input image features to all the features that I have formed from the training images. Voting based on multiple nearest neighbors instead of taking the closest match improved my accuracy by 4%. I believe that this parameter was best tuned to this specific set, and does not generally hold for all image sets.

SVM Classify involves using a support vector machine to train 15 classifiers for the 15 different categories. This was a linear SVM meaning that the feature space was partitioned into a learned plane and the test images are sorted based on which side of the hyperplane they fall in (i.e., it either matches or doesn't). I input my test image into each of the 15 classifiers and then categorized the image based on which had the highest score of matching. SVM Classify is a stronger classification method than nearest neighbor because it downplays the uninformative patches and only focuses on those which are important. I improved performance by utilizing matrix multiplication rather than loops to determine scores for the 1 vs. all. That is, I used repmat to get the parameters used for determining scores in a matrix and then performed matrix multiplication. I also found that a lambda parameter of .00001 led to the best accuracy.

Results

I had three different combinations of representation and classification with varying levels of accuracy:

  1. Tiny images and nearest neighbor with 21.7% accuracy
  2. Bag of SIFT representation and nearest neighbor with 55.1% accuracy
  3. Bag of SIFT Representation and linear SVM with 68.8% accuracy
